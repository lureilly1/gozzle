**gozzle** is a local safety harness for ClickHouse, exposed to your AI client
over [MCP](https://modelcontextprotocol.io). The AI reasons; gozzle runs bounded
checks and returns **verdicts plus proof**.

gozzle ships as `@gozzle/cli`, which provides a `gozzle` CLI and a `gozzle-mcp`
stdio server.

## How it differs from the official ClickHouse MCP

The official ClickHouse MCP is a query gateway: it lets the model run SELECTs and
returns rows. gozzle is a **safety harness**:

- Every query runs **read-only by construction** ([read-only enforcement](/docs/read-only)).
- Every query is **bounded** so the model can't trigger a runaway ([cost & row guardrails](/docs/guardrails)).
- Checks return **ClickHouse-aware verdicts** the model can't compute from raw
  rows, like [`verify_dedup`](/docs/verify-dedup).
- Every call can be **recorded locally** for review ([audit log](/docs/audit-log)).

## Tools

| Tool | What it does |
| --- | --- |
| `health` | Confirm the gozzle MCP server is running. |
| `connect` | Validate the connection and report read-only guardrails and account privileges. |
| `inspect_table` | Report a table's physical layout, parts, keys, and eligible checks. |
| [`verify_dedup`](/docs/verify-dedup) | Prove whether a ReplacingMergeTree table currently holds duplicate rows by sorting key. |
| [`verify_equivalent`](/docs/verify) | Prove two SELECTs return the same result — exact, computed in ClickHouse. |
| [`create_local_slice`](/docs/local-slices) | Copy one complete partition to a bounded, persistent local chDB workspace and compare source/local proof. |
| [`dry_run_migration`](/docs/dry-run-migration) | Classify one ALTER statement and estimate touched rows, parts, and bytes without executing it. |
| [`diagnose_query`](/docs/diagnose-query) | Prove partition and primary-key pruning behavior with EXPLAIN, with separate query-shape advisories. |

## Install

**Project-local (recommended for teams)** — the version is committed to the repo,
so everyone (and CI) runs the same gozzle:

```bash
npm install -D @gozzle/cli
npx gozzle init --local      # prints an MCP config that launches via npx
```

**Global** — for a quick personal setup:

```bash
npm install -g @gozzle/cli
gozzle init
```

Add the printed config to Claude, Cursor, Codex, or another MCP host. Use a
read-only ClickHouse user.
